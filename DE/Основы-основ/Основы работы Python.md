В Python переменная — это не "ячейка памяти" как в C++, а **имя (name)**, которое привязано к **объекту (object)** в памяти. Грубо говоря, переменная — это ярлык, наклеенный на объект.

- **Объект** — это область памяти, которая содержит:
    
    1. Самые данные (значение).
        
    2. Счетчик ссылок (сколько на него "наклеено ярлыков").
        
    3. Тип данных (который тоже является объектом и неизменяем).
        
- **Имя (переменная)** — это просто символьное имя (ярлык), которое ссылается на этот объект.
    

**Простая аналогия:** Представьте шкаф с ящиками (объекты в памяти). На ящик можно наклеить несколько стикеров с разными именами (переменные). Переменная — это не сам ящик, а лишь стикер с именем, указывающий на него.

---

### **1. Присваивание и идентичность**

Оператор присваивания `=` не копирует объект, а всего лишь **связывает имя с объектом** в памяти.

```python
a = [1, 2, 3]  # Создали объект-список [1,2,3] в памяти и наклеили на него ярлык "a"
b = a          # Наклеили ВТОРОЙ ярлык "b" на ТОТ ЖЕ САМЫЙ объект.
```
**Как это проверить? Оператор `is`.**

Он проверяет, ссылаются ли два имени на **один и тот же объект** в памяти (идентичность).
```python
print(a is b)  # True. Оба имени указывают на один объект.
print(id(a) == id(b)) # True. id() возвращает уникальный идентификатор объекта.
```
**Следствие:**  
Любое изменение объекта через одно имя будет видно через все другие имена, которые на него ссылаются.
```python
a.append(4)
print(a)  # [1, 2, 3, 4]
print(b)  # [1, 2, 3, 4] <- b тоже "увидело" изменение, т.к. это тот же объект!
```
### **2. Изменяемость (Mutability) и её последствия**

Это прямое следствие модели "ссылок". Поведение при изменении объекта зависит от его типа.

#### **Изменяемые (mutable) типы: `list`, `dict`, `set`**

Их содержимое можно изменить "на месте" (in-place), не создавая новый объект.

```python
list_1 = [1, 2, 3]
list_2 = list_1   # Обе переменные ссылаются на один список

list_1[0] = 999   # Изменяем объект "на месте"

print(list_1)     # [999, 2, 3]
print(list_2)     # [999, 2, 3] <- изменилась тоже!
```

#### **Неизменяемые (immutable) типы: `int`, `float`, `str`, `tuple`, `frozenset`**

Их состояние изменить нельзя. Любая операция, которая выглядит как изменение, на самом деле создает **новый объект**.

```python
a = "hello"
b = a        # Обе переменные ссылаются на один объект-строку "hello"

a = a + " world"  # Не изменяем "hello", а создаем НОВУЮ строку "hello world"
                  # и перенаправляем ярлык 'a' на этот новый объект.

print(a)     # "hello world" (новый объект)
print(b)     # "hello"      (старый объект, на который все еще ссылается 'b')
```

### **3. Передача аргументов в функции (Call by Assignment)**

В Python аргументы в функцию всегда передаются **по присваиванию** (call by object reference). Это значит, что в функцию передается не копия объекта, а ссылка на тот же самый объект.

Поведение внутри функции зависит от типа переданного объекта:

- **Если объект неизменяемый:** Вы не можете изменить оригинальный объект. Любая попытка "изменения" создаст новый локальный объект внутри функции.

```python
def try_change_immutable(x):
    x = x + 10  # Создается новый объект, локальная ссылка 'x' перенаправляется на него
    print("Inside func:", x)

num = 5
try_change_immutable(num)
print("Outside:", num) # Оригинал не изменился
# Output:
# Inside func: 15
# Outside: 5
```

**Если объект изменяемый:** Вы можете изменить его содержимое "на месте", и это изменение затронет оригинал.

```python
def change_mutable(some_list):
    some_list.append("modified") # Изменяем объект на месте

my_list = [1, 2, 3]
change_mutable(my_list)
print(my_list) # [1, 2, 3, 'modified'] <- Оригинал изменен!
```

### **4. Хеширование (Hashability)**

**Хешируемый объект** — это объект, который имеет хеш-значение (целое число), неизменное на протяжении всего времени его жизни.

- **Неизменяемые объекты (int, str, tuple, frozenset)** обычно хешируемы.
    
    - У `tuple` есть нюанс: он хешируем только если хешируемы все его элементы. `(1, 2, [3, 4])` — не хешируем, т.к. содержит список.
        
- **Изменяемые объекты (list, dict, set)** — нехешируемы. Их хеш не может быть постоянным, так как их содержимое может измениться.
    

**Зачем это нужно?**  
Хешируемость — это необходимое условие для того, чтобы объект мог быть **ключом в словаре (`dict`)** или **элементом множества (`set`)**. Эти структуры данных используют хеш-значение для сверхбыстрого поиска и проверки на уникальность.

```python
# Работает
valid_dict = { 
    "name": "Alice",       # str (immutable)
    42: "Answer",          # int (immutable)
    (1, 2): "tuple value"  # tuple (immutable)
}

# Не работает (вызовет TypeError: unhashable type)
invalid_dict = {
    [1, 2]: "list key"    # list (mutable)
    {1, 2}: "set key"     # set (mutable)
}
```
### **Итог для инженера данных:**

1. **Все есть объект и ссылка на него.** Понимание этого избавит от тонны багов, связанных с неожиданным изменением данных.
2. **Различай изменяемое и неизменяемое.** Для передачи данных, которые не должны меняться, используй кортежи (`tuple`) или `frozenset`. Для конфигураций — неизменяемые структуры.
3. **При работе с изменяемыми объектами помни о копировании.** Если тебе нужна именно независимая копия списка или словаря, используй:
    - `.copy()` для поверхностного копирования.
    - `copy.deepcopy()` для полного (глубокого) копирования вложенных структур.
4. **Функции могут менять свои аргументы.** Передавая в функцию изменяемый объект, ты даешь ей право изменить его. Если это нежелательно — передавай копию.
5. **Хешируемость — ключ к словарям.** Ключами в словарях и элементами множеств могут быть только неизменяемые (хешируемые) типы. Это основа для быстрого поиска, который постоянно используется в обработке данных (например, при `JOIN`-ах в Pandas).
