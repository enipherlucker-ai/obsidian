**Функциональное выражение** — это способ определить функцию «на лету», как значение выражения, без использования классического оператора `def`. В Python основной способ создания функциональных выражений — **лямбда-выражения** (`lambda`).

Синтаксис:

```python
lambda параметры: выражение
```

Ключевые особенности:
- Тело lambda — ровно **одно выражение** (не может содержать блоков `if/else` как statement, циклов `for/while`, `return`, `yield`). Тернарный оператор допустим: `lambda x: x if x > 0 else -x`.
- Результат выражения автоматически возвращается — писать `return` нельзя и не нужно.
- Lambda создаёт **анонимную функцию** — у неё нет имени (точнее, имя внутри `__name__` будет `'<lambda>'`).
- Lambda можно присвоить переменной, но PEP 8 рекомендует в таком случае использовать `def`.

```python
# Обычная функция
def square(x):
    return x ** 2

# Эквивалент через lambda
square_lambda = lambda x: x ** 2

print(square(5))         # 25
print(square_lambda(5))  # 25
```

**Когда использовать lambda:**
- Короткие одноразовые функции, передаваемые как аргумент (`key=`, `map()`, `filter()`).
- Ситуации, где определять `def` излишне.

**Когда НЕ использовать lambda:**
- Сложная логика (больше одного выражения).
- Когда функцию нужно переиспользовать — лучше `def` с понятным именем.

#### 1.2 Функции первого класса (first-class functions)

В Python функции являются **объектами первого класса** (first-class citizens). Это значит, что функция — такой же объект, как число, строка или список, и с ней можно делать всё то же самое:

1. **Присваивать переменной:**

```python
def greet(name):
    return f"Привет, {name}!"

hello = greet           # hello теперь ссылается на ту же функцию
print(hello("Мир"))     # "Привет, Мир!"
print(type(hello))      # <class 'function'>
```

2. **Хранить в структурах данных:**

```python
operations = {
    'add': lambda a, b: a + b,
    'sub': lambda a, b: a - b,
    'mul': lambda a, b: a * b,
}
print(operations['add'](3, 5))  # 8
```

3. **Передавать как аргумент** другой функции (подробнее ниже).

4. **Возвращать из функции** как результат (подробнее ниже).

5. **Иметь атрибуты** — у функций в Python есть `__name__`, `__doc__`, `__defaults__`, `__code__` и т.д.

#### 1.3 Использование функций как аргументов

Передача функции как аргумента другой функции — фундаментальный приём функционального программирования. Функция-аргумент часто называется **callback**.

**Встроенные примеры:**

```python
numbers = [3, 1, 4, 1, 5, 9, 2, 6]

# sorted() с key= — функция, определяющая «по чему сортировать»
sorted_nums = sorted(numbers)                     # [1, 1, 2, 3, 4, 5, 6, 9]
sorted_by_dist = sorted(numbers, key=lambda x: abs(x - 5))  # по расстоянию до 5

# map() — применяет функцию к каждому элементу
squares = list(map(lambda x: x ** 2, numbers))    # [9, 1, 16, 1, 25, 81, 4, 36]

# filter() — оставляет элементы, для которых функция возвращает True
evens = list(filter(lambda x: x % 2 == 0, numbers))  # [4, 2, 6]

# min()/max() с key=
data = [('Иван', 30), ('Мария', 25), ('Пётр', 35)]
youngest = min(data, key=lambda person: person[1])  # ('Мария', 25)
```

**Свой пример с callback:**

```python
def apply_to_list(lst, func):
    """Применяет функцию func к каждому элементу списка."""
    return [func(item) for item in lst]

result = apply_to_list([1, 2, 3, 4], lambda x: x * 10)
print(result)  # [10, 20, 30, 40]
```

**В pandas:**

```python
import pandas as pd
df = pd.DataFrame({'price': [100, 200, 300]})
df['discounted'] = df['price'].apply(lambda p: p * 0.9)
df['category'] = df['price'].map(lambda p: 'cheap' if p < 200 else 'expensive')
```

#### 1.4 Функции высшего порядка (higher-order functions)

**Функция высшего порядка** — это функция, которая выполняет хотя бы одно из двух:
- **Принимает** другую функцию как аргумент;
- **Возвращает** функцию как результат.

Функция, которая не делает ни того, ни другого — это функция **первого порядка**.

##### Примеры встроенных функций высшего порядка

| Функция | Что делает |
|---------|------------|
| `map(func, iterable)` | Применяет `func` к каждому элементу |
| `filter(func, iterable)` | Оставляет элементы, где `func` вернула True |
| `sorted(iterable, key=func)` | Сортирует по результату `func` |
| `functools.reduce(func, iterable)` | Последовательно «сворачивает» коллекцию |

```python
from functools import reduce

# reduce складывает элементы попарно: ((1+2)+3)+4 = 10
total = reduce(lambda acc, x: acc + x, [1, 2, 3, 4])
```

##### Функция, возвращающая функцию

```python
def make_multiplier(factor):
    """Возвращает функцию, умножающую аргумент на factor."""
    def multiplier(x):
        return x * factor
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(10))  # 20
print(triple(10))  # 30
```

Здесь `make_multiplier` — функция высшего порядка: она **возвращает** новую функцию. Переменная `factor` «запоминается» внутри возвращённой функции — это **замыкание (closure)**.

##### Функция, принимающая и возвращающая функцию — декоратор

```python
import time

def timer(func):
    """Декоратор: измеряет время выполнения функции."""
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} заняла {elapsed:.4f} сек")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)

slow_function()  # slow_function заняла 1.00xx сек
```

Декоратор `timer` принимает функцию `func` и возвращает обёртку `wrapper`.

#### 1.5 Сопутствующие вопросы

**Что такое замыкание (closure)?**

Замыкание — функция, которая «захватывает» переменные из окружающей области видимости. Даже когда внешняя функция завершилась, внутренняя продолжает иметь доступ к этим переменным.

```python
def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

c = counter()
print(c())  # 1
print(c())  # 2
print(c())  # 3
```

**Чем `map()`/`filter()` отличаются от list comprehension?**

Функционально эквивалентны, но list comprehension считается более «pythonic» и часто читабельнее:

```python
# map + lambda
list(map(lambda x: x ** 2, range(10)))
# list comprehension
[x ** 2 for x in range(10)]
```

Однако передача именованной функции в `map()` может быть лаконичнее:
```python
list(map(str.upper, ['hello', 'world']))  # ['HELLO', 'WORLD']
```

**Что такое `functools.partial`?**

`partial` фиксирует часть аргументов функции, возвращая новую функцию:

```python
from functools import partial

def power(base, exponent):
    return base ** exponent

square = partial(power, exponent=2)
cube = partial(power, exponent=3)

print(square(5))  # 25
print(cube(5))    # 125
```
