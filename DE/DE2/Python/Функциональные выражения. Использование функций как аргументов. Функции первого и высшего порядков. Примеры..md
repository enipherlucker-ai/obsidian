## Доклад

В Python функции являются полноценными объектами. Это означает, что их можно сохранять в переменные, передавать в другие функции, возвращать из функций и использовать почти так же, как числа, строки или списки. Именно на этом строится идея **функций первого порядка** и **функций высшего порядка**.

Если начать с функциональных выражений, то чаще всего под этим имеют в виду использование функции как значения внутри выражения, а также создание небольших анонимных функций через `lambda`.  
Например, мы можем не только объявить обычную функцию через `def`, но и записать короткую функцию прямо в выражении:

``` python
square = lambda x: x * x  
print(square(5))  # 25
```

Здесь `lambda x: x * x` — это функциональное выражение, которое создаёт функцию без имени.  
Обычно такой подход используют там, где логика очень короткая и нужна один раз: например, при сортировке, фильтрации или преобразовании коллекций.

Важно понимать, что `lambda` — это не какой-то особый тип функции, а просто сокращённая форма записи небольшой функции.  
То есть такой код:
```python
def square(x):  
    return x * x
```
и такой код:
``` python
square = lambda x: x * x
```
по смыслу очень близки.

## Использование функций как аргументов

Одна из сильных сторон Python — возможность передавать функцию в другую функцию как аргумент.  
То есть мы можем сказать: “вот данные, а вот правило обработки этих данных”.

Например:
``` python
def apply_func(x, func):  
    return func(x)  
  
def double(n):  
    return n * 2  
  
print(apply_func(10, double))  # 20
```

Здесь `double` передаётся как аргумент в `apply_func`.  
Это удобно, потому что сама функция `apply_func` становится универсальной: она не знает заранее, что именно делать с числом, а просто вызывает ту функцию, которую ей передали.

Это особенно полезно в Data Engineering, потому что в пайплайнах часто нужно строить **универсальные шаги обработки**:  
например, одна и та же обвязка может применять разные правила очистки, валидации, нормализации или преобразования данных.

## Функции первого порядка

Когда говорят, что в Python есть **функции первого класса** или **функции первого порядка**, имеют в виду, что функции можно:

- присваивать переменным;
- хранить в структурах данных;
- передавать как аргументы;
- возвращать из других функций.

Пример с присваиванием в переменную:
``` python
def greet(name):  
    return f"Hello, {name}"  
  
say_hello = greet  
print(say_hello("Alice"))  # Hello, Alice
```

Здесь переменная `say_hello` ссылается на ту же функцию, что и `greet`.

Пример со списком функций:
``` python
def add_one(x):  
    return x + 1  
  
def multiply_by_two(x):  
    return x * 2  
  
operations = [add_one, multiply_by_two]  
  
for op in operations:  
    print(op(10))
```
Такой подход полезен, когда мы хотим хранить набор преобразований и применять их по очереди.

## Функции высшего порядка

**Функция высшего порядка** — это функция, которая либо:

1. принимает другую функцию как аргумент,  
либо
2. возвращает функцию как результат.

То есть не каждая функция первого порядка — это функция высшего порядка.  
Функции первого порядка — это свойство языка в целом.  
А функция высшего порядка — это уже конкретная функция, которая работает с другими функциями.

Пример функции высшего порядка, которая принимает функцию как аргумент, мы уже видели:
``` python
def apply_func(x, func):  
    return func(x)
```
Пример функции высшего порядка, которая возвращает другую функцию:
``` python
def make_multiplier(n):  
    def multiplier(x):  
        return x * n  
    return multiplier  
  
double = make_multiplier(2)  
triple = make_multiplier(3)  
  
print(double(5))  # 10  
print(triple(5))  # 15
```
Здесь `make_multiplier` создаёт и возвращает новую функцию.  
Это удобно, когда мы хотим заранее “настроить” поведение функции.

## Встроенные примеры функций высшего порядка в Python

В Python есть несколько стандартных примеров такого подхода:

### 1. `map()`

Применяет функцию к каждому элементу коллекции.
``` python
numbers = [1, 2, 3, 4]  
result = list(map(lambda x: x * 2, numbers))  
print(result)  # [2, 4, 6, 8]
```
### 2. `filter()`

Оставляет только те элементы, для которых функция возвращает `True`.
``` python
numbers = [1, 2, 3, 4, 5, 6]  
result = list(filter(lambda x: x % 2 == 0, numbers))  
print(result)  # [2, 4, 6]
```
### 3. `sorted()` с параметром `key`

Очень частый практический случай.
``` python
users = [  
    {"name": "Alice", "age": 30},  
    {"name": "Bob", "age": 25},  
    {"name": "Charlie", "age": 35}  
]  
  
result = sorted(users, key=lambda user: user["age"])  
print(result)
```
Здесь в `key` передаётся функция, которая объясняет, по какому признаку сортировать.

## Где это применяется в Data Engineering

Для Data Engineer эта тема важна не только теоретически, но и практически.

### 1. Гибкая обработка данных

Можно строить универсальные функции, в которые передаётся конкретная логика преобразования.
``` python
def transform_column(values, transform_func):  
    return [transform_func(v) for v in values]  
  
data = ["  apple ", " banana  ", " cherry "]  
cleaned = transform_column(data, lambda x: x.strip().upper())  
  
print(cleaned)  # ['APPLE', 'BANANA', 'CHERRY']
```
Одна и та же функция `transform_column` может применять любые правила.

---

### 2. Переиспользуемые шаги в ETL/ELT

Можно передавать валидаторы, парсеры, конвертеры типов.
``` python
def process_records(records, validator):  
    return [r for r in records if validator(r)]  
  
records = [  
    {"id": 1, "amount": 100},  
    {"id": 2, "amount": -10},  
    {"id": 3, "amount": 50}  
]  
  
valid = process_records(records, lambda r: r["amount"] >= 0)  
print(valid)
```
Такой подход делает код модульным и удобным для тестирования.

---

### 3. Кастомизация поведения

Например, можно передать функцию логирования, стратегию обработки ошибок или правило агрегации.

---

## Плюсы такого подхода

Основные преимущества:

- код становится более гибким;
- уменьшается дублирование;
- проще переиспользовать общую логику;
- легче тестировать маленькие функции по отдельности;
- удобнее собирать пайплайн из независимых шагов.

---

## На что обратить внимание

Хотя функциональный стиль полезен, злоупотреблять им не стоит.

Например, слишком сложные `lambda` ухудшают читаемость.  
Если логика длинная или содержит несколько условий, лучше использовать обычную `def`.

Плохо:
``` python
result = list(map(lambda x: x.strip().lower() if x and len(x) > 3 else "unknown", values))
```
Лучше вынести в именованную функцию:
``` python
def normalize_value(x):  
    if x and len(x) > 3:  
        return x.strip().lower()  
    return "unknown"
```
То есть на практике хороший стиль — использовать `lambda` только для очень коротких и очевидных операций.

## Короткий вывод

Итого:  
в Python функции являются объектами первого класса, поэтому их можно передавать, сохранять и возвращать.  
На этом строятся функции высшего порядка — функции, которые принимают другие функции или возвращают их.  
Это даёт гибкость, позволяет писать универсальные инструменты обработки данных и активно используется в реальных ETL- и Data Engineering-задачах: при трансформации, фильтрации, сортировке, валидации и построении пайплайнов.

**В Python функции — это объекты первого класса. Их можно передавать как аргументы, сохранять в переменные и возвращать из других функций. # Вопросы, которые могут задать

### 1. Что значит, что функция — объект первого класса?

Это значит, что функцию можно присвоить переменной, передать как аргумент, вернуть из другой функции и хранить в коллекциях.

### 2. Чем функция первого порядка отличается от функции высшего порядка?

Функции первого порядка — это общее свойство языка: функции можно использовать как объекты.  
Функция высшего порядка — это конкретная функция, которая принимает другую функцию или возвращает её.

### 3. Что такое `lambda`?

Это способ создать небольшую анонимную функцию в одну строку. Обычно используется для коротких операций.

### 4. Когда лучше использовать `lambda`, а когда `def`?

`lambda` — для простых и коротких выражений.  
`def` — когда логика сложнее, нужна читаемость, документация или повторное использование.

### 5. Какие встроенные функции Python относятся к функциям высшего порядка?

`map`, `filter`, `sorted`, иногда `reduce` из модуля `functools`.

### 6. Где это встречается в работе Data Engineer?

В ETL/ELT, при очистке данных, валидации, кастомной сортировке, применении разных преобразований к колонкам и строкам.

### 7. Почему это удобно?

Потому что позволяет отделить общую механику обработки данных от конкретной логики преобразования.

### 8. Есть ли минусы?

Да. Если слишком увлечься функциональным стилем, код может стать трудно читаемым, особенно при сложных `lambda` и вложенных вызовах.
