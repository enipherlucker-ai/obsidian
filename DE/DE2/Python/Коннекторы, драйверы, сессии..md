#### 3.1 Что такое драйвер БД

**Драйвер** — низкоуровневая библиотека, реализующая сетевой протокол конкретной СУБД. Драйвер устанавливает TCP-соединение, выполняет аутентификацию, сериализует/десериализует запросы и результаты в бинарном формате СУБД.

В Python стандарт API для драйверов — **DB-API 2.0** (PEP 249). Он определяет:
- Функцию `connect()` → объект Connection
- Метод `connection.cursor()` → объект Cursor
- Методы курсора: `execute()`, `executemany()`, `fetchone()`, `fetchmany()`, `fetchall()`
- Стандартные исключения: `DatabaseError`, `IntegrityError`, `OperationalError` и т.д.
- Управление транзакциями: `connection.commit()`, `connection.rollback()`

Примеры драйверов:

| СУБД | Драйвер | Особенности |
|------|---------|-------------|
| PostgreSQL | `psycopg2` | Обёртка над libpq (C), быстрый, проверенный |
| PostgreSQL | `psycopg3` (psycopg) | Новая версия, async, pipeline mode |
| PostgreSQL | `asyncpg` | Чистый async, максимальная скорость |
| MySQL | `mysql-connector-python` | Официальный от Oracle |
| MySQL | `PyMySQL` | Чистый Python, проще установка |
| MSSQL | `pyodbc` | Через ODBC, универсальный |
| MSSQL | `pymssql` | Через FreeTDS |
| ClickHouse | `clickhouse-driver` | Нативный протокол ClickHouse |
| ClickHouse | `clickhouse-connect` | HTTP-интерфейс |
| MongoDB | `pymongo` | Официальный драйвер |
| SQLite | `sqlite3` | Встроен в стандартную библиотеку |

Пример работы с драйвером:
```python
import psycopg2

conn = psycopg2.connect(
    host='localhost', port=5432,
    dbname='mydb', user='user', password='pass'
)
try:
    with conn.cursor() as cur:
        cur.execute("SELECT id, name FROM users WHERE age > %s", (18,))
        rows = cur.fetchall()
        for row in rows:
            print(row)
    conn.commit()
finally:
    conn.close()
```

#### 3.2 Что такое коннектор

**Коннектор** — более общий термин, обозначающий компонент, обеспечивающий подключение к внешней системе. Коннектор может быть:

- **Синонимом драйвера** (например, MySQL Connector/Python).
- **Высокоуровневой абстракцией**, оборачивающей драйвер. Например, SQLAlchemy Engine — коннектор, который использует драйвер psycopg2 внутри.
- **Компонентом ETL-системы**: Kafka Connect, Airbyte Connector, Fivetran Connector — готовые модули для чтения/записи данных из/в конкретный источник.
- **Объектом подключения в оркестраторе**: в Airflow `Connection` — запись с host, port, login, password, extra, которую используют операторы и хуки.

Иерархия:
```
Приложение → Коннектор (SQLAlchemy / Airflow Hook) → Драйвер (psycopg2) → TCP → СУБД
```

#### 3.3 Что такое сессия (Session)

**Сессия** — логический контекст взаимодействия с БД, в рамках которого сохраняется состояние: активная транзакция, серверные настройки (SET), временные таблицы, prepared statements.

**Connection vs Session:**
- **Connection** — физический TCP-канал до БД. Создание дорого (TCP handshake, аутентификация, выделение памяти на сервере).
- **Session** — логический контекст поверх Connection. В PostgreSQL 1 connection = 1 session. В некоторых СУБД можно мультиплексировать.

**Пул соединений (Connection Pool):**

Создавать соединение на каждый запрос дорого. Пул заранее создаёт N соединений и выдаёт их по запросу:

```python
from psycopg2 import pool

connection_pool = pool.ThreadedConnectionPool(
    minconn=2,
    maxconn=10,
    host='localhost', dbname='mydb', user='user', password='pass'
)

conn = connection_pool.getconn()
try:
    with conn.cursor() as cur:
        cur.execute("SELECT 1")
finally:
    connection_pool.putconn(conn)  # вернуть в пул, а не закрыть

connection_pool.closeall()
```

**SQLAlchemy Session:**

В SQLAlchemy Session реализует паттерн **Unit of Work** — отслеживает изменения объектов и применяет их одним commit:

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

engine = create_engine(
    'postgresql+psycopg2://user:pass@localhost/mydb',
    pool_size=5,       # минимальное число соединений в пуле
    max_overflow=10,    # сколько дополнительных можно создать
    pool_recycle=3600   # пересоздавать соединения старше 1 часа
)
Session = sessionmaker(bind=engine)

with Session() as session:
    user = session.query(User).filter_by(id=1).first()
    user.name = 'Новое имя'
    session.commit()  # все изменения отправляются одной транзакцией
```

#### 3.4 Сопутствующие вопросы

**Что такое ODBC и JDBC?**
- **ODBC** (Open Database Connectivity) — стандартный интерфейс доступа к БД для C/C++. Python использует через `pyodbc`. Универсален: один код для разных СУБД, если установлен ODBC-драйвер.
- **JDBC** (Java Database Connectivity) — аналог для Java. Python может использовать через `JayDeBeApi`.

**Зачем нужен connection pool?**
- Избегает накладных расходов на создание/уничтожение TCP-соединений.
- Ограничивает число одновременных соединений к БД (БД имеет лимит: `max_connections` в PostgreSQL).
- Обеспечивает переиспользование «прогретых» соединений.

**Что такое курсор (Cursor)?**
Cursor — объект, через который выполняются запросы и извлекаются результаты. Cursor хранит указатель на текущую позицию в результирующем наборе (result set). Серверные курсоры (server-side) позволяют читать данные порциями без загрузки всего результата в память.
