## Доклад

При работе с Python и внешними системами очень часто приходится взаимодействовать с API по протоколу HTTP.  
Самые распространённые типы запросов — это **GET** и **POST**.

Если говорить просто, то **GET** обычно используется, чтобы **получить данные с сервера**, а **POST** — чтобы **отправить данные на сервер** для создания или обработки.

То есть основное различие такое:

- **GET** — запрашивает информацию;
- **POST** — отправляет информацию.

---

## Что такое HTTP-запрос

Когда Python-приложение обращается к API, оно отправляет HTTP-запрос на определённый URL.  
Запрос содержит:

- адрес ресурса;
- метод запроса, например GET или POST;
- заголовки;
- иногда параметры;
- иногда тело запроса.

Сервер принимает этот запрос, обрабатывает его и возвращает ответ:  
обычно это статус-код, заголовки и тело ответа, часто в формате JSON.

---

## GET-запрос

**GET-запрос** используется для чтения данных.  
Например, мы можем запросить список пользователей, получить информацию по заказу или забрать данные из внешнего сервиса.

Особенность GET в том, что параметры чаще всего передаются прямо в URL.

Пример:
```python
import requests  
  
response = requests.get(  
    "https://api.example.com/users",  
    params={"page": 1, "limit": 10}  
)  
  
print(response.status_code)  
print(response.json())
```
Здесь Python отправляет GET-запрос на API и передаёт параметры `page` и `limit`.  
В итоге реальный адрес может выглядеть примерно так:
```
https://api.example.com/users?page=1&limit=10
```
GET-запросы обычно используют, когда нужно:

- получить справочник;
- забрать список записей;
- запросить конкретный объект по ID;
- получить данные для дальнейшей загрузки в пайплайн.

---

## POST-запрос

**POST-запрос** используется, когда нужно передать данные на сервер.  
Чаще всего это создание новой сущности, запуск обработки или отправка полезной нагрузки в API.

В отличие от GET, данные у POST обычно передаются не в URL, а в **теле запроса**.

Пример:
``` python
import requests  
  
payload = {  
    "name": "Alice",  
    "email": "alice@example.com"  
}  
  
response = requests.post(  
    "https://api.example.com/users",  
    json=payload  
)  
  
print(response.status_code)  
print(response.json())
```
Здесь мы отправляем JSON в теле POST-запроса.  
Сервер может создать нового пользователя и вернуть результат.

POST часто используют, когда нужно:

- создать запись;
- отправить данные на обработку;
- авторизоваться;
- инициировать задачу;
- передать пакет данных в API.

---

## Основное различие между GET и POST

Если кратко, то разница такая:

### 1. Назначение

- **GET** — получение данных;
- **POST** — отправка данных.

### 2. Где передаются параметры

- у **GET** — обычно в URL;
- у **POST** — обычно в теле запроса.

### 3. Идемпотентность и безопасность

GET считается безопасным методом чтения: он не должен изменять состояние сервера.  
POST, наоборот, может изменять состояние, например создавать новую запись.

### 4. Кэширование

GET-запросы легче кэшируются, потому что они предназначены для чтения.  
POST обычно не кэшируется так же просто.

---

## Как это выглядит в Python

На практике в Python чаще всего используют библиотеку `requests`.

### GET:

import requests  
  
response = requests.get("https://api.example.com/orders/123")  
data = response.json()  
print(data)

### POST:

import requests  
  
payload = {"order_id": 123, "status": "processed"}  
response = requests.post("https://api.example.com/orders/update", json=payload)  
print(response.json())

---

## Статус-коды ответа

При работе с GET и POST важно смотреть не только на данные, но и на **HTTP status code**.

Наиболее частые коды:

- **200 OK** — запрос выполнен успешно;
    
- **201 Created** — объект успешно создан;
    
- **400 Bad Request** — ошибка в параметрах запроса;
    
- **401 Unauthorized** — нет авторизации;
    
- **403 Forbidden** — доступ запрещён;
    
- **404 Not Found** — ресурс не найден;
    
- **500 Internal Server Error** — ошибка на стороне сервера.
    

Пример проверки:

import requests  
  
response = requests.get("https://api.example.com/users/1")  
  
if response.status_code == 200:  
    print(response.json())  
else:  
    print(f"Ошибка: {response.status_code}")

## Заголовки и авторизация

Очень часто API требует передавать заголовки, например токен доступа.

Пример:

import requests  
  
headers = {  
    "Authorization": "Bearer my_token",  
    "Content-Type": "application/json"  
}  
  
response = requests.get(  
    "https://api.example.com/data",  
    headers=headers  
)  
  
print(response.status_code)

Для Data Engineer это особенно важно, потому что интеграции с API почти всегда требуют авторизацию.

---

## Где это применяется в Data Engineering

Для Data Engineer GET и POST запросы — это практический рабочий инструмент.

### 1. Получение данных из внешних API

Например, через GET можно забирать:

- курсы валют;
    
- данные клиентов;
    
- информацию о заказах;
    
- события из внешней системы;
    
- справочники.
    

### 2. Отправка данных в другую систему

Через POST можно:

- отправлять результаты обработки;
    
- публиковать данные в сервис;
    
- создавать задачи на внешней платформе;
    
- вызывать запуск обработки.
    

### 3. Интеграции в ETL/ELT

В ETL-процессе один шаг может делать GET и забирать данные,  
а следующий — трансформировать их и через POST отправлять в downstream-систему.

### 4. Работа с оркестраторами и служебными API

Например, пайплайн может POST-запросом запускать задачу или уведомление,  
а GET-запросом проверять её статус.

---

## Практический пример для Data Engineer

Представим, что у нас есть пайплайн, который забирает заказы из внешнего сервиса.

Сначала мы делаем GET-запрос:

import requests  
  
response = requests.get("https://api.example.com/orders", params={"date": "2026-03-15"})  
orders = response.json()

После этого обрабатываем данные внутри Python.  
А затем можем отправить агрегированный результат в другую систему через POST:

import requests  
  
result = {  
    "date": "2026-03-15",  
    "total_orders": 125,  
    "total_amount": 54000  
}  
  
response = requests.post("https://api.example.com/report", json=result)

То есть GET — это входящий поток данных,  
а POST — исходящий поток данных или команда на действие.

---

## На что обращать внимание в реальной работе

При использовании GET и POST в продакшене важно помнить несколько вещей.

### 1. Обработка ошибок

Нельзя слепо считать, что API всегда отвечает успешно.  
Нужно проверять статус-код и корректно логировать ошибки.

### 2. Таймауты

Запрос может зависнуть, поэтому лучше явно задавать timeout.

Пример:

import requests  
  
response = requests.get("https://api.example.com/data", timeout=10)

### 3. Повторы запросов

Иногда API временно недоступно, и нужна retry-логика.

### 4. Ограничения API

У многих сервисов есть rate limit — ограничение на число запросов.

### 5. Валидация ответа

Нужно проверять, что сервер вернул именно те поля, которые мы ожидаем.

---

## Хорошая практика в Python

Часто пишут не просто один запрос, а оборачивают вызовы API в функции.

Пример:

import requests  
  
def get_users(page: int):  
    response = requests.get(  
        "https://api.example.com/users",  
        params={"page": page},  
        timeout=10  
    )  
    response.raise_for_status()  
    return response.json()

Так код становится чище, проще тестируется и переиспользуется.

# Вопросы, которые могут задать

### 1. В чём главное различие между GET и POST?

GET используется для получения данных, POST — для отправки данных на сервер.

### 2. Где передаются параметры у GET и POST?

У GET — чаще в URL, у POST — обычно в теле запроса.

### 3. Какую библиотеку чаще всего используют в Python для HTTP-запросов?

Обычно используют библиотеку `requests`.

### 4. Что такое `response.json()`?

Это метод, который преобразует JSON-ответ сервера в Python-словарь или список.

### 5. Какие статус-коды нужно знать?

Минимум: 200, 201, 400, 401, 403, 404, 500.

### 6. Почему важно ставить timeout?

Чтобы приложение не зависло бесконечно при проблемах на стороне сети или сервера.

### 7. Где это используется в Data Engineering?

При интеграции с внешними API, загрузке данных, отправке результатов обработки, запуске задач и проверке статусов.

### 8. Может ли GET изменять данные на сервере?

По стандартной логике — не должен. GET предназначен для чтения.

### 9. Когда использовать POST вместо GET?

Когда нужно отправить данные, создать объект или инициировать действие на сервере.

### 10. Что важно делать в проде при работе с API?

Проверять статус-коды, обрабатывать ошибки, ставить timeout, учитывать retry и rate limits.
