### 10.1. Определение Big O

**Big O** (O-нотация) описывает **верхнюю границу** скорости роста функции при стремлении размера входных данных к бесконечности:

> f(n) = O(g(n)), если существуют константы c > 0 и n₀ ≥ 0 такие, что для всех n ≥ n₀: f(n) ≤ c · g(n).

Другими словами, O(g(n)) — это множество функций, которые растут **не быстрее**, чем g(n) с точностью до константного множителя.

### 10.2. Правила упрощения

1. **Отбросить константы:** O(3n) → O(n), O(1000) → O(1).
2. **Отбросить младшие слагаемые:** O(n² + n + 42) → O(n²).
3. **Логарифмы:** O(log₂ n) = O(log₁₀ n) = O(ln n) — основание не важно (отличаются на константу).

### 10.3. Основные классы сложности

| Обозначение | Название | Пример |
|-------------|----------|--------|
| **O(1)** | Константная | Доступ по индексу `list[i]`, `dict[key]`, `len()` |
| **O(log n)** | Логарифмическая | Бинарный поиск (`bisect`), поиск в B-Tree |
| **O(n)** | Линейная | Проход по списку, `sum()`, `max()`, `x in list` |
| **O(n log n)** | Линейно-логарифмическая | Сортировка (Timsort), merge sort |
| **O(n²)** | Квадратичная | Вложенные циклы, bubble sort, naive string matching |
| **O(n³)** | Кубическая | Умножение матриц (наивное), Floyd–Warshall |
| **O(2ⁿ)** | Экспоненциальная | Полный перебор подмножеств, наивная рекурсия Фибоначчи |
| **O(n!)** | Факториальная | Перебор всех перестановок (задача коммивояжёра brute-force) |

Скорость роста: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

### 10.4. Амортизированная сложность

Отдельная операция может быть дорогой, но **в среднем** по серии операций — дешёвой.

**Пример — `list.append()`:**
- Обычно O(1) — просто записать элемент в выделенный буфер.
- Иногда O(n) — буфер заполнен, нужно выделить новый (×1.125 от текущего размера) и скопировать все элементы.
- **Амортизированно** каждый `append` — **O(1)**.

**Пример — `dict`:**
- Вставка обычно O(1), но при превышении 2/3 заполненности таблица пересоздаётся (rehash) — O(n). Амортизированно: O(1).

### 10.5. Пространственная сложность (Space Complexity)

Описывает, сколько **дополнительной** памяти требует алгоритм:

```python
# O(1) по памяти — на месте
def reverse_inplace(lst):
    left, right = 0, len(lst) - 1
    while left < right:
        lst[left], lst[right] = lst[right], lst[left]
        left += 1
        right -= 1

# O(n) по памяти — создаём копию
def reverse_copy(lst):
    return lst[::-1]
```

### 10.6. Сложность встроенных структур данных Python

**list (динамический массив):**

| Операция | Средняя | Худшая |
|----------|---------|--------|
| `list[i]` (доступ по индексу) | O(1) | O(1) |
| `list.append(x)` | O(1) аморт. | O(n) |
| `list.insert(0, x)` | O(n) | O(n) |
| `list.pop()` (с конца) | O(1) | O(1) |
| `list.pop(0)` (с начала) | O(n) | O(n) |
| `x in list` | O(n) | O(n) |
| `list.sort()` | O(n log n) | O(n log n) |
| `list[i:j]` (срез) | O(j − i) | O(j − i) |

**dict (хеш-таблица):**

| Операция | Средняя | Худшая |
|----------|---------|--------|
| `dict[key]` | O(1) | O(n) |
| `dict[key] = val` | O(1) аморт. | O(n) |
| `key in dict` | O(1) | O(n) |
| `del dict[key]` | O(1) | O(n) |
| Итерация | O(n) | O(n) |

**set (хеш-множество):**

| Операция | Средняя | Худшая |
|----------|---------|--------|
| `x in set` | O(1) | O(n) |
| `set.add(x)` | O(1) | O(n) |
| `set.union(other)` | O(len(s) + len(t)) | — |
| `set.intersection(other)` | O(min(len(s), len(t))) | — |

**deque (двусторонняя очередь):**

| Операция | Сложность |
|----------|-----------|
| `deque.append(x)` | O(1) |
| `deque.appendleft(x)` | O(1) |
| `deque.pop()` | O(1) |
| `deque.popleft()` | O(1) |
| `deque[i]` (доступ по индексу) | O(n) |

### 10.7. Хорошие практики

**1. Используйте `set` / `dict` для проверки принадлежности:**

```python
# ПЛОХО — O(n) на каждую проверку, итого O(n * m)
blacklist = [1, 2, 3, ..., 100_000]
for item in data:  # len(data) = m
    if item in blacklist:
        ...

# ХОРОШО — O(1) на каждую проверку, итого O(n + m)
blacklist = set(blacklist)  # O(n) один раз
for item in data:
    if item in blacklist:   # O(1)
        ...
```

**2. Используйте `bisect` для бинарного поиска:**

```python
import bisect

sorted_list = list(range(1_000_000))

# ПЛОХО — O(n)
idx = sorted_list.index(999_999)

# ХОРОШО — O(log n)
idx = bisect.bisect_left(sorted_list, 999_999)
```

**3. Конкатенация строк — `str.join()` вместо `+=`:**

```python
# ПЛОХО — O(n²): каждая += создаёт новую строку и копирует предыдущую
result = ""
for word in words:
    result += word + " "

# ХОРОШО — O(n): join вычисляет суммарную длину, выделяет буфер один раз
result = " ".join(words)
```

**4. `deque.appendleft()` вместо `list.insert(0, x)`:**

```python
from collections import deque

# ПЛОХО — O(n) на каждую вставку → O(n²) суммарно
lst = []
for i in range(100_000):
    lst.insert(0, i)

# ХОРОШО — O(1) на каждую вставку → O(n) суммарно
dq = deque()
for i in range(100_000):
    dq.appendleft(i)
```

### 10.8. Плохие практики

**1. Вложенные циклы без необходимости:**

```python
# ПЛОХО — O(n²): поиск пересечения через вложенные циклы
common = []
for x in list_a:
    for y in list_b:
        if x == y:
            common.append(x)

# ХОРОШО — O(n + m): пересечение множеств
common = list(set(list_a) & set(list_b))
```

**2. Проверка `x in list` внутри цикла:**

```python
# ПЛОХО — O(n * m)
for x in big_list:
    if x in another_big_list:
        process(x)

# ХОРОШО — O(n + m)
lookup = set(another_big_list)
for x in big_list:
    if x in lookup:
        process(x)
```

### 10.9. Примеры из Data Engineering

**Hash Join vs Nested Loop Join:**

```python
# Nested Loop Join — O(n * m)
for row_a in table_a:
    for row_b in table_b:
        if row_a['id'] == row_b['id']:
            yield {**row_a, **row_b}

# Hash Join — O(n + m)
index = {}
for row_b in table_b:          # O(m) — строим хеш-индекс
    index[row_b['id']] = row_b

for row_a in table_a:           # O(n) — пробиваем по индексу
    if row_a['id'] in index:    # O(1) lookup
        yield {**row_a, **index[row_a['id']]}
```

**Почему Spark/SQL-движки выбирают Hash Join:** при больших таблицах O(n + m) кратно быстрее O(n · m). Broadcast Hash Join в Spark рассылает маленькую таблицу на все узлы.
