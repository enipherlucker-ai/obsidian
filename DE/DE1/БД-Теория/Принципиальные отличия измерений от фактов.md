## 1. Определения

- **Факты (Facts)** — количественные показатели бизнес-процессов, которые можно измерить, агрегировать и анализировать (например: сумма продаж, количество заказов).
- **Измерения (Dimensions)** — справочные данные, описывающие контекст фактов и позволяющие группировать или фильтровать показатели (например: дата, клиент, товар, регион).

## 2. Основные характеристики

| Критерий                  | Факты                                            | Измерения                                              |
| ------------------------- | ------------------------------------------------ | ------------------------------------------------------ |
| **Назначение**            | Хранение числовых показателей, метрик            | Описание атрибутов и контекста фактов                  |
| **Тип данных**            | Обычно числовые (целые, вещественные, суммы)     | Чаще категориальные или текстовые                      |
| **Примеры**               | Выручка, количество продаж, себестоимость        | Дата, товар, магазин, клиент, менеджер                 |
| **Гранулярность**         | Конкретный уровень детализации (строка заказа)   | Многоуровневые атрибуты (иерархия: год → месяц → день) |
| **Возможность агрегации** | Да (SUM, AVG, COUNT, MIN, MAX)                   | Нет (но можно группировать по измерениям)              |
| **Изменяемость**          | Постоянно пополняются новыми событиями           | Изменяются редко (изменение справочников)              |
| **Объем данных**          | Очень большие (миллионы и миллиарды строк)       | Сравнительно небольшие (тысячи, десятки тысяч)         |
| **Связь**                 | Ссылаются на измерения через ключи (foreign key) | Используются как справочники (primary key)             |
## 3. Пример

- **Факт**: "Продажа товара" → количество = 5, сумма = 10 000 ₽.
- **Измерения**: дата = «2025-09-16», товар = «Ноутбук», магазин = «Москва».

---
## 4. Взаимосвязь фактов и измерений

- Факты **без измерений** не имеют смысла («продажи = 100» — непонятно чего, когда и где).
- Измерения **без фактов** неинтересны для анализа (справочник товаров без данных о продажах).
- Вместе они образуют **звездообразную схему (Star Schema)** или «снежинку», используемую в хранилищах данных.

---
## 5. Ключевая мысль для аттестации

- **Факты = “что произошло и в каком объеме”**.
- **Измерения = “в каком контексте это произошло”**.
- Инженер данных должен уметь правильно выделять факты и измерения для проектирования моделей данных (DWH, OLAP).
