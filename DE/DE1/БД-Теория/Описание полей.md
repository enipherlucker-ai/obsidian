#### **1. Описание полей (Типы данных)**

Это фундамент, определяющий, какие данные и как будут храниться. Выбор типа критически важен для эффективности хранения и скорости запросов.

**Общее для всех СУБД:**

- **Целочисленные:** `TINYINT`, `SMALLINT`, `INT`/`INTEGER`, `BIGINT`. **Важно:** сразу использовать `BIGINT` для ключей в больших таблицах во избежание переполнения.
- **Числа с плавающей точкой:** `FLOAT`, `DOUBLE`. Нельзя использовать для точных расчетов (деньги).
- **Числа с фиксированной точкой:** `DECIMAL`/`NUMERIC`(p,s) — для точных расчетов (финансы).
- **Строковые:** `VARCHAR(n)`, `TEXT`(неограниченный).
- **Дата и время:** `DATE`, `TIME`, `DATETIME`/`TIMESTAMP`.
    
**Особенности ClickHouse:**  
ClickHouse предлагает богатейший набор типов для оптимизации под конкретные задачи:

- **Высокая специфичность:** `IPv4`, `IPv6`, `UUID`, `Enum8`, `Enum16`.

- **Эффективность для массивов и вложенных структур:**
    - `Array(T)` — массив элементов типа `T`.
    - `Tuple(T1, T2, ...)` — кортеж.
    - `Nested` — вложенные структуры (по сути, `Array` + `Tuple`).
    - `Map(key, value)` — словарь (ассоциативный массив).

- **Специальные типы для аналитики:**
    - `AggregateFunction` — хранит промежуточное состояние агрегатной функции (например, для `uniqCombined`).
    - `LowCardinality(T)` — **Ключевой тип для оптимизации.** Позволяет эффективно хранить и обрабатывать строковые поля с малым количеством уникальных значений (enum-like). Резко увеличивает производительность.
    - **Домены:** `Domain` — пользовательские типы на основе существующих (например, `URL` на основе `String`).

**Вывод:** В ClickHouse инженер данных должен не просто объявить тип, а **выбрать наиболее эффективный** для конкретного сценария (`LowCardinality`, домены, специфичные типы).
