Определяет, как физически организованы данные на диске и как с ними работает СУБД.

**Классические СУБД (OLTP):**  
Структура обычно следует стандарту SQL: `CREATE TABLE (column1 type, column2 type, ...)`.

**ClickHouse:**  
Структура таблицы неразрывно связана с ее **движком (Engine)**. Помимо колонок, в `CREATE TABLE` обязательно указываются:
1. **Движок таблицы (Engine):** Определяет логику хранения и основные возможности (см. раздел 4).
2. **Ключ сортировки (ORDER BY):** **Важнейшая настройка.** Определяет, в каком порядке данные физически лежат на диске. От этого зависит эффективность `WHERE`, `GROUP BY`. **Не гарантирует уникальность!** Это не `PRIMARY KEY` в классическом понимании.
3. **Первичный ключ (PRIMARY KEY):** В ClickHouse — это всего лишь подмножество ключа сортировки, используемое для индексации. Он не обеспечивает уникальность.
4. **Партиционирование (PARTITION BY):** Опционально. Разбивает данные на логические части (например, по дате). Ускоряет операции с данными по партициям (DROP, ATTACH).

**Пример структуры в ClickHouse:**

```sql
CREATE TABLE logs (
    event_date Date,
    event_time DateTime,
    user_id UInt32,
    url String,
    browser LowCardinality(String)
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(event_date)
ORDER BY (event_date, user_id) -- Ключ сортировки
PRIMARY KEY (event_date) -- Первичный ключ (опционально, обычно часть ORDER BY)
```
