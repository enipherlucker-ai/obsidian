## 1. Определения

- **OLTP (Online Transaction Processing)** — системы онлайн-транзакционной обработки. Предназначены для выполнения большого количества коротких операций (транзакций) с данными.
    
- **OLAP (Online Analytical Processing)** — системы онлайн-аналитической обработки. Используются для анализа больших объёмов данных, построения отчетов, прогнозов и аналитических витрин.

## 2. Основные характеристики

|Критерий|OLTP|OLAP|
|---|---|---|
|**Назначение**|Обработка текущих бизнес-операций|Анализ и агрегирование данных для принятия решений|
|**Тип нагрузки**|Частые небольшие транзакции (INSERT/UPDATE/DELETE)|Сложные аналитические запросы (SELECT, JOIN, GROUP BY)|
|**Объем данных**|Меньше, хранение актуальных данных|Очень большие объемы (история за годы)|
|**Модель данных**|Нормализованная (3NF) для исключения дублирования|Денормализованная (звезда, снежинка) для скорости|
|**Характер запросов**|Простые, быстрые, на уровне строк|Сложные, многоуровневые, с агрегацией|
|**Время отклика**|Миллисекунды|Секунды, минуты, иногда часы|
|**Пользователи**|Операторы, кассиры, бухгалтеры (операционный персонал)|Аналитики, руководители, BI-системы|
|**Актуальность**|Данные «здесь и сейчас»|Исторические данные, тенденции|
|**Примеры**|ERP, CRM, банковские системы, e-commerce|DWH (хранилище данных), BI-системы, аналитические кубы|
## 3. Пример аналогии

- **OLTP** — «касса магазина» (операции покупки прямо сейчас).
- **OLAP** — «отчёт директора магазина за год» (анализ продаж, прогнозы).

---
## 4. Взаимосвязь OLTP и OLAP

- Данные обычно сначала поступают в OLTP-системы.
- Затем они реплицируются, очищаются и агрегируются в хранилище (ETL/ELT процесс).
- OLAP использует эти данные для аналитики и отчетности.

---
## 5. Ключевая мысль для аттестации

- **OLTP = скорость транзакций и консистентность данных.**
- **OLAP = глубина анализа и производительность агрегаций.**
- В инженерной практике они дополняют друг друга: OLTP обслуживает бизнес-операции, OLAP помогает принимать стратегические решения.
