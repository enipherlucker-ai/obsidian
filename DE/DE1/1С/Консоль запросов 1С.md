### 🔹 Что это?

- **Консоль запросов 1С** – это встроенный инструмент (обработка), который используется для написания, отладки и выполнения запросов к данным в базе 1С.
    
- Она работает с **языком запросов 1С (СУБД-независимый язык)**, который трансформируется системой в SQL под конкретную СУБД (PostgreSQL, MS SQL, Oracle и т. д.).
    

---

### 🔹 Для чего нужна?

1. Проверка и отладка запросов без изменения конфигурации.
    
2. Анализ данных из справочников, документов, регистров и т.д.
    
3. Эксперименты с выборками, агрегатами, временными таблицами.
    
4. Ускорение разработки: можно быстро проверить логику выборки, а потом перенести готовый запрос в код.
    

---

### 🔹 Чем отличается от запросов в обычных БД?

| В 1С (Консоль запросов)                                                                | В обычных БД (SQL)                                |
| -------------------------------------------------------------------------------------- | ------------------------------------------------- |
| Используется **специальный язык запросов 1С**                                          | Используется SQL                                  |
| Запросы пишутся к **объектам метаданных** (Справочник.Товары, РегистрСведений.Продажи) | Запросы пишутся к **таблицам/представлениям**     |
| Автоматическая подстановка нужных таблиц и связей                                      | Нужно явно указывать все таблицы и связи          |
| Независимость от СУБД – один и тот же запрос работает в разных СУБД                    | Запрос зависит от конкретной СУБД                 |
| Есть специфические конструкции 1С: Период, СрезПоследних, СрезПервых и др.             | Только стандартные SQL-операторы                  |
| Учитываются особенности платформы (версии данных, права пользователей, уровни доступа) | SQL работает напрямую с данными без бизнес-логики |

---

👉 Итог: **Консоль запросов 1С** – это инструмент разработчика для проверки и отладки выборок на языке запросов 1С. В отличие от обычного SQL, она работает не напрямую с таблицами, а через объекты метаданных и учитывает специфику платформы (версии данных, регистры, права).
