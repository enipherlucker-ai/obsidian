#### **1. Основной вопрос: Что такое правило LEGB и как оно работает?**

**Правило LEGB** — это набор правил, по которым интерпретатор Python ищет значение переменной или функции по её имени. Это аббревиатура, описывающая порядок и иерархию областей видимости (scopes), в которых происходит поиск:

**L — Local (Локальная)**

- **Что это:** Область видимости внутри функции или метода.
    
- **Поиск:** Переменные, объявленные внутри функции с помощью оператора присваивания (`=`).
    
- **Пример:**
    
```python
def func():
    local_var = "I'm local"  # Объявлена внутри func -> Local scope
    print(local_var)

func()  # Вывод: I'm local
# print(local_var)  # NameError: name 'local_var' is not defined
```
**E — Enclosing (Охватывающая / Nonlocal)**

- **Что это:** Область видимости внешних функций (для вложенных функций). Это не глобальная область, а область функции-родителя.
    
- **Поиск:** Переменные из тела внешней функции, внутри которой объявлена текущая.
    
- **Пример:**
```python    
def outer_func():
    enclosing_var = "I'm from enclosing scope"  # Объявлена в outer_func -> Enclosing scope

    def inner_func():
        # inner_func может "видеть" переменные из outer_func
        print(enclosing_var)  # Найдено в Enclosing scope

    inner_func()

outer_func()  # Вывод: I'm from enclosing scope
```
**G — Global (Глобальная)**

- **Что это:** Область видимости на уровне модуля (файла `.py`).
    
- **Поиск:** Переменные, объявленные вне всех функций или помеченные ключевым словом `global`.
    
- **Пример:**
    
```python 
global_var = "I'm global"  # Объявлена на уровне модуля -> Global scope

def func():
    print(global_var)  # Найдено в Global scope

func()  # Вывод: I'm global
```
**B — Built-in (Встроенная)**

- **Что это:** Самая внешняя область. Содержит встроенные функции и исключения Python (например, `print`, `len`, `ValueError`).
    
- **Поиск:** Происходит в последнюю очередь, если имя не найдено ни в одной из предыдущих областей.
    
- **Пример:**
    
```python 
def func():
    # print не объявлен ни локально, ни в enclosing, ни глобально.
    # Интерпретатор найдет его во встроенной области (Built-in).
    print("Hello, World!")

func()  # Вывод: Hello, World!
```
**Порядок поиска:** Интерпретатор ищет имя **последовательно, от L к B**: `Local -> Enclosing -> Global -> Built-in`. Поиск прекращается, как только имя находится.

#### **2. Ключевые слова `global` и `nonlocal`**

Правило LEGB описывает поиск для **чтения** значений. Для **изменения** переменных из внешних областей используются специальные ключевые слова.

**`global`** — позволяет изменять переменную из глобальной области видимости.

```python

counter = 0  # Global

def increment():
    global counter  # Явно говорим Python, что работаем с ГЛОБАЛЬНОЙ переменной counter
    counter += 1
    print(f"Counter inside: {counter}")

increment()  # Counter inside: 1
increment()  # Counter inside: 2
print(f"Counter outside: {counter}")  # Counter outside: 2
```
_Без `global` инструкция `counter += 1` создала бы новую локальную переменную и вызвала бы ошибку `UnboundLocalError`._

**`nonlocal`** — позволяет изменять переменную из ближайшей охватывающей (enclosing) области видимости (но не глобальной).

```python

def outer():
    count = 0  # Enclosing scope для inner

    def inner():
        nonlocal count  # Явно говорим Python, что работаем с переменной из ENCLOSING области
        count += 1
        return count

    return inner

counter = outer()
print(counter())  # 1
print(counter())  # 2
```
_Это классический пример замыкания (closure), где внутренняя функция запоминает состояние переменной из внешней._

#### **3. Возможные дополнительные вопросы и ответы на них**

**1. Что произойдет, если объявить переменную с тем же именем, что и во внешней области?**

- **Ответ:** Будет создана новая локальная переменная, которая **затенит** (shadow) переменную из внешней области. Изменение локальной переменной не затронет внешнюю.
    
```python
x = "global"

def func():
    x = "local"  # Создается новая локальная переменная, а не меняется глобальная
    print(x)      # Вывод: local

func()
print(x)          # Вывод: global (глобальная переменная не изменилась)
```
**2. В чем разница между `nonlocal` и `global`?**

- **Ответ:** `nonlocal` ищет переменную в цепочке охватывающих (enclosing) областей, но **не доходит до глобальной**. `global` сразу переходит на уровень модуля. `nonlocal` используется только внутри вложенных функций.
    

**3. Всегда ли поиск по правилу LEGB такой медленный?**

- **Ответ:** Нет, для локальных переменных Python использует очень быстрые оптимизации (например, хранит их в массиве, доступ по индексу). Самое медленное — обращение к встроенным именам (Built-in), но их кеширование на уровне байт-кода в современных версиях Python (CPython 3.11+) минимизирует эту затрату.
    

**4. Как правило LEGB связано с замыканиями (closures)?**

- **Ответ:** Замыкание — это функция, которая запоминает значения из охватывающей области видимости (Enclosing scope), даже когда эта область завершила свою работу. Правило LEGB — это механизм, который делает замыкания возможными, позволяя внутренней функции находить и использовать переменные из внешней функции после её выполнения.

#### **4. Особенности и "подводные камни"**

1. **Изменение mutable объектов без `global`/`nonlocal`:**
    
    - **Камень:** Правило LEGB работает только на **присваивание** (`=`). Если вы изменяете содержимое **изменяемого** (mutable) объекта (например, списка или словаря), полученного из внешней области, ключевые слова не нужны и не используются.
        
    - **Пример:**
        
```python
my_list = []  # Global mutable object

def append_to_list(item):
    my_list.append(item)  # Не присваивание! Мы меняем внутреннее состояние объекта.
    # my_list = [item]   # А вот это было бы присваивание и создало бы локальную переменную!

append_to_list(1)
print(my_list)  # [1] - объект изменен
```
- - **Решение:** Четко понимать разницу между изменением состояния объекта и перепривязкой имени.
        
- **`UnboundLocalError` — самый частый камень:**
    
    - **Камень:** Переменная считается локальной, если ей хоть раз присваивали значение в теле функции. Если вы попытаетесь прочитать её значение до присваивания, возникнет ошибка.
        
    - **Пример:**
        
```python
x = 10

def func():
    print(x)  # UnboundLocalError!
    x = 20    # Т.к. есть присваивание, интерпретатор считает 'x' локальной переменной.

func()
```
- - **Решение:** Использовать `global` или `nonlocal`, если вы хотели работать с внешней переменной, либо быть внимательным к порядку операций.
        
- **Замыкания и позднее связывание (Late Binding):**
    
    - **Камень:** В цикле часто создаются несколько замыканий, которые используют одну и ту же переменную из охватывающей области. К моменту их вызова цикл уже завершен, и все они используют **последнее** значение переменной цикла.
        
    - **Пример:**
        
```python
functions = []
for i in range(3):
    def func():
        return i  # Все функции будут помнить переменную 'i' из Enclosing scope
    functions.append(func)

# i теперь равна 2
for f in functions:
    print(f())  # Все выводят 2, а не 0, 1, 2 как можно было бы ожидать!
```
**Решение:** Использовать аргументы по умолчанию (которые вычисляются сразу) для захвата текущего значения:

python

for i in range(3):
    def func(x=i):  # Значение i захватывается в момент создания функции
        return x
    functions.append(func)

Это критически важно для написания корректного, предсказуемого и эффективного кода, особенно при построении сложных ETL-конвейеров и архитектур приложений.
