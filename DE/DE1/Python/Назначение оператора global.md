`global` — это ключевое слово в Python, которое позволяет **внутри функции ссылаться на переменную из глобальной области видимости** (а не создавать локальную).

По умолчанию любая переменная, объявленная в функции, считается **локальной**, даже если глобальная переменная с таким же именем существует.  
Чтобы изменить именно глобальную переменную, нужно использовать `global`.

## 🔹 Почему это хорошо

1. **Простота** — легко и быстро изменить глобальную переменную.
     ^b7da0a
2. **Удобство для маленьких скриптов** — например, когда нужна одна «глобальная настройка» или «счётчик».
    
3. **Можно избежать передачи переменных через аргументы функций**, если они общие для всей программы.
    

---

## 🔹 Почему это плохо

1. **Запутывает код** — становится неочевидно, где и кем изменяется переменная.
    
2. **Скрытые зависимости** — функция зависит от внешнего состояния, что усложняет отладку и тестирование.
    
3. **Проблемы при масштабировании** — в больших проектах глобальные переменные приводят к ошибкам и конфликтам.
    
4. **Нарушение принципа «функция = чёрный ящик»** — функция должна работать только с входными данными, а не менять что-то снаружи.
    

✅ Допустимо (маленькие скрипты, счётчики, быстрые прототипы) ^9e551a
```python
import time

running = True

def stop():
    global running
    running = False

while running:
    print("Работаю...")
    time.sleep(1)
    stop()  # остановим после первого вызова
```

❌ Плохая практика (глобальные настройки)
```python
debug = True

def log(msg):
    if debug:
        print(msg)

def toggle_debug():
    global debug
    debug = not debug
```
