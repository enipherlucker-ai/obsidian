**Лямбда-функция (анонимная функция)** – это способ создать небольшую, одноразовую функцию без использования ключевого слова `def` и без явного указания имени. Она определяется на лету в месте использования.

**Синтаксис:**
```python
lambda arguments: expression
```
Ключевые особенности:

- Ключевое слово `lambda`.
    
- Принимает любое количество аргументов.
    
- Выполняет одно _выражение_ (`expression`) и автоматически возвращает его результат.
    
- Не содержит блоков кода, циклов или операторов `return`.
    

**Польза и применение:**  
Их основная польза – **лаконичность** и **удобство** в ситуациях, когда:

1. Функция используется всего один раз в коде.
    
2. Функция настолько проста, что ее определение через `def` загромождает код.
    
3. Функция требуется как аргумент для другой функции (чаще всего в комбинации с функциями `map()`, `filter()`, `sorted()`, и особенно – в методах pandas).
    

#### **Применение с pandas.DataFrame**

В контексте работы с данными в pandas лямбда-функции наиболее часто и полезно применяются в комбинации с методами:

- `apply()` – применение функции вдоль оси DataFrame или Series.
    
- `applymap()` – применение функции к каждому элементу DataFrame (для поэлементных операций).
    
- `map()` – подстановка значений в Series на основе другой функции или словаря.
    
- `agg()` / `aggregate()` – агрегация данных с помощью одной или нескольких операций.
    
- `sort_values()` – кастомная сортировка.
    
- `assign()` – создание новых колонок через вычисления.
    

**Ключевые сценарии использования в pandas:**

**1. Создание новой колонки на основе преобразования существующих:**
```python
import pandas as pd

df = pd.DataFrame({'price': [100, 200, 150], 'quantity': [1, 3, 2]})

# Добавляем колонку 'total' = price * quantity
df['total'] = df.apply(lambda row: row['price'] * row['quantity'], axis=1)
```

**2. Сложное преобразование значений в колонке:**

```python
df = pd.DataFrame({'email': ['ivanov@gmail.com', 'petrov@yahoo.com']})

# Извлекаем домен из email-адреса
df['domain'] = df['email'].apply(lambda x: x.split('@')[-1])
```

**3. Фильтрация строк DataFrame:**  
Часто используется с `apply`, но для простых условий лучше использовать булеву индексацию.

```python
# Оставляем строки, где в названии города больше 5 букв
df = pd.DataFrame({'city': ['Moscow', 'SPb', 'Novosibirsk']})
filtered_df = df[df['city'].apply(lambda x: len(x) > 5)]
```

**4. Агрегация с пользовательской логикой:**

```python
# Хотим найти размах (max - min) для каждой числовой колонки
df.agg(lambda x: x.max() - x.min())
```

**5. Сортировка по сложному правилу:**

```python
df = pd.DataFrame({'name': ['Alice Smith', 'Bob Johnson', 'Charlie Brown']})

# Сортируем по фамилии (последнему слову в строке)
df_sorted = df.sort_values(by='name', key=lambda col: col.str.split().str[-1])
```

### **2. Дополнительные вопросы и ответы на них**

**1. В чем главное преимущество лямбда-функций перед обычными?**  
Главное преимущество – **лаконичность**. Они позволяют не объявлять отдельную функцию с помощью `def` для простых, одноразовых операций, делая код более читаемым _в месте применения_. Это устраняет необходимость перескакивать взглядом к другому месту в коде (например, к блоку с `def`).

**2. Всегда ли лямбда-функция – лучший выбор?**  
**Нет.** Если логика становится сложной (больше одного выражения, нужны условия `if-elif-else`, циклы), использование обычной функции `def` или определение функции отдельно всегда предпочтительнее. Лямбда-функции предназначены для простых операций.

**3. Можно ли использовать в лямбде условные операторы?**  
Да, но только тернарный оператор `(value_if_true) if (condition) else (value_if_false)`.

```python
df['price_category'] = df['price'].apply(
    lambda p: 'Expensive' if p > 100 else 'Cheap'
)
```

**4. Чем `apply` с лямбдой отличается от векторных операций в pandas?**

- **`apply` с лямбдой:** Это, по сути, **обход элементов в цикле** (на уровне Python). Медленно для больших датасетов.
    
- **Векторные операции:** Используют оптимизированные, скомпилированные низкоуровневые процедуры библиотек (NumPy, C++). Работают **на порядки быстрее**.
    

**Вывод для DE:** Всегда старайтесь сначала найти встроенный векторный метод pandas (например, `str.methods`, `dt.methods`, арифметические операции), и только если готового решения нет, используйте `apply` с лямбдой.

### **3. Особенности и "подводные камни"**

1. **Производительность – главный камень!** Чрезмерное использование `apply(lambda x: ...)` на больших DataFrame – **верный путь к медленному коду**. Это самая частая и критичная ошибка. Всегда спрашивайте себя: "Можно ли это сделать через встроенные векторные операции?".
    
    - **Плохо (медленно):** `df['col'].apply(lambda x: x + 1)`
        
    - **Хорошо (быстро):** `df['col'] + 1`
        
2. **Читаемость.** Слишком сложная лямбда-функция, растянутая на много строк или с вложенными условиями, ухудшает читаемость кода. Помните: "Код читается чаще, чем пишется". Если логика неочевидна, выносите ее в отдельную функцию с понятным именем.
    
3. **Отладка.** Анонимные функции сложнее отлаживать, так как в трассировке стека (traceback) будет фигурировать общее имя `<lambda>`, а не понятное имя функции. Это может затруднить поиск места ошибки.
    
4. **Ограниченность функциональности.** В лямбде можно записать только одно выражение. Невозможно использовать операторы `return`, `pass`, `assert` или циклы внутри самой лямбды.
    

**Заключение для инженера данных:**  
Лямбда-функции – это **мощный инструмент для быстрого прототипирования и написания лаконичного кода** для простых преобразований данных в pandas. Однако Senior-инженер всегда должен помнить об их **производительностных издержках** и использовать их обдуманно, отдавая приоритет встроенным векторным операциям pandas/NumPy для обработки больших данных.
